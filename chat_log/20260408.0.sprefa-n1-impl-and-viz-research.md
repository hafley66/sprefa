### Vision Discussion
User's goal: an Armored Core garage-style UI for coding. Save a file in VS Code, see the blast radius across all repos on disk as a spatial graph. Repos as contained groups ("papers behind papers"), files as nodes, refs as edges. Layout engine as a separate crate that outputs JSON positions, renderer is swappable (egui, TUI, VS Code webview).

Architecture: scan tables (SQLite) -> layout crate (Rust, computes positions) -> renderer (draws it)

## Key Insights
- The compound layout gap is a crate worth writing -- CoSE paper is public, algorithm is a modified force-directed sim with containment forces
- Two-level force-directed is the workaround until compound layout exists: layout per container, then layout containers relative to each other
- For incremental "on save" updates, hot-start full re-layout is fast enough for blast-radius-scoped graphs (tens to hundreds of nodes)
- Fjädra (Rerun's internal force layout) is pure Rust, no deps, WASM-compatible -- worth investigating as a base
- fdg (222 stars) and forceatlas2 are the active Rust force-directed options
- keyframe (easing/bezier) and spanda (spring physics) cover animation math
