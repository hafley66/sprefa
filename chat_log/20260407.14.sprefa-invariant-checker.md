# Session: sprefa-invariant-checker

## Goal
Build an invariant checking system for sprefa using raw SQL check blocks. Design: `check(name) { SQL };` blocks parsed from .sprf files, executed after scans, with violations stored in `invariant_violations` table for later querying.

## Current State
Partially implemented. Schema changes complete (migrations, views with norm/norm2 columns). AST and parsing complete. Store trait methods added. SQLite implementation has duplicate code bug that needs fixing. Check execution not yet wired into scanner. CLI command not yet added.

## Problem/Context
User wants check blocks to use existing per-rule views that join on strings. Views must expose `norm` and `norm2` columns from the strings table for normalized comparisons. Rows returned from SQL queries = violations to be stored and queried later.
