# Session: sprefa-group-id-query-redesign

## Goal
Implement the group_id foundation for the query redesign plan (crystalline-finding-dolphin.md). The broader plan redesigns the .sprf query layer: rule heads declare output shape, group_id couples co-extracted captures, rule names become queryable relations, check construct with negation.

## Current State
group_id on matches table is implemented and committed (42cdbdb). Schema, flush, RawRef, emit, extractor all wired up. 249 non-watch tests pass. Watch test failures are pre-existing (ON CONFLICT vs conditional unique index).

## Problem/Context
The query redesign plan (crystalline-finding-dolphin.md) has multiple layers. User scoped this session to just the DB/core layer -- group_id on matches -- deferring AST/parser and query compiler changes. User explicitly rejected lowercasing variable names to derive kinds ("i hate renames, that was bot suggestion that survived") -- kinds stay as-is (SCREAMING case, same as the variable name, no rename step).

## Solution/Approach
- group_id INTEGER column on matches table with index
- RawRef.group: Option<u32> field threads group identity from extraction to flush
- RuleExtractor assigns per-file group counter, increments per MatchResult (one extraction site = one group)
- Built-in extractors (js, rs) pass group: None
- Flush maps (file_id, local_group) to globally unique group_ids via MAX(group_id) + offset
- ALTER TABLE migration for existing DBs

## Key Insights
- User directive: kinds = variable name as-is, no lowercasing rename step. $SVC produces kind "SVC" not "svc".
- User directive: no AST/parser this session, just DB and core scan pipeline.
- The group boundary is the MatchResult boundary from the walk engine. One walk result = one group.
- Per-file group tags (u32 counter in extractor) get mapped to global group_ids during flush via MAX(group_id) offset. This avoids needing cross-file coordination.
- repo()/rev() in rule heads replace IS_REPO/IS_REV match() annotations -- same semantics, better syntax. Underlying match_labels machinery unchanged.
