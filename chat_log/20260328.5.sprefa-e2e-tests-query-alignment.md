### sprefa_query schema alignment
- 0_types.rs: Hit.ref_kind: RefKind replaced with kind: String + rule_name: String. Filter::OfKind/NotKind take Vec<String>.
- 1_eval.rs: atom SQL LEFT JOINs matches, selects COALESCE(m.kind, ''), COALESCE(m.rule_name, ''). Filters compare string kinds.
- 2_standing.rs: StandingRule gains kind: String field. Uses rule_name TEXT column not rule_id FK. Batch ref_id resolution via OR-chained WHERE instead of N+1. get_matches queries WHERE m.rule_name = ? directly.

## Key Insights
- parent_key on emitted refs comes from the "parent" field in emit config, not from the structural position. cargo-deps dep_name has no parent; package-json-deps dep_version has parent: "name".
- SQLite doesn't support JOIN (VALUES ...) AS v(col1, col2). Use OR-chained WHERE clauses for batch lookups.
- Standing rule hash-change invalidation: upsert_rule deletes all old matches when hash differs, then evaluate_rule re-inserts. Stale removal via diff only works when hash is unchanged.
- Committed flush is additive for branch_files -- doesn't remove old refs when files change. Standing rule stale detection depends on eval returning fewer hits, not on refs being cleaned up.
- User confirmed: index is droppable, no backwards compat needed. Just make the code correct.
- User direction on URTSL: don't build the parser/language. Build the core Rust execution layer. Time/git/PR stuff comes later via ghcacher companion library.
