# Session: sprefa-kill-matches-table

## Goal
Eliminate the `matches` table by migrating built-in extractors (JS, RS) to use per-rule tables, unifying the two parallel storage paths.

## Current State
Plan approved, implementation not started. BOARD.md updated. Plan file at `~/.claude/plans/dynamic-marinating-sphinx.md`.

Partial cleanup already applied (not committed):
- Removed `match_links` DELETE from flush.rs
- Removed `group_id` column from matches DDL in migrations.rs
- Updated flush.rs comment

## Problem/Context
Two parallel storage paths exist:
1. Built-in extractors (JS/RS) -> `refs + matches` (polymorphic kind column)
2. .sprf rule extractors -> per-rule `_data` tables (typed columns)

`matches` is a free-floating relation table doing the same job as per-rule tables but worse. Also found: dead `flush_repo_meta` (references removed `repo_refs` table), broken `search_refs` second UNION arm, dead `resolve_links` stub.

## Solution/Approach
10 built-in kind tables (import_path, import_name, import_alias, export_name, export_local_binding, dep_name, dep_version, rs_use, rs_declare, rs_mod), each with columns (value, lang). `lang` is a plain TEXT column for future multi-language support.

4 phases:
1. Infrastructure: `builtin_rule_table_defs()` in rule_tables.rs, register at startup
2. Flip extractors: rule_name = kind, group = Some(n), so to_file_results includes them
3. Rewrite consumers: queries.rs (10 functions), watcher.rs, resolve.rs, search_refs, flush.rs
4. Remove dead code: drop matches DDL, delete flush_repo_meta, delete resolve_links, fix tests

## Key Insights
- `matches` table is heavily used by watch/rewrite system (queries.rs, watcher.rs) -- not dead, just redundant with per-rule tables
- Built-in extractors set `group: None` which excludes them from per-rule tables via `to_file_results()`. Setting group to Some(n) is the key unlock.
- `flush_repo_meta` references removed `repo_refs` table -- entirely dead code
- `search_refs` second UNION arm references removed `repo_ref_id` column -- broken
- `lang` column enables future cross-language semantic grouping without table restructuring
- Context forks (`^level`) discussion punted, not part of this work
- Pattern re-use (`let` bindings for regex) discussion punted
