## Key Discovery
ast-grep with `--lang json` and `--lang yaml` captures key-value pairs with metavariables:

```yaml
# This rule captures all dep key-value pairs from JSON
id: capture-dep-kv
language: json
rule:
  pattern:
    context: '{"$KEY": "$VAL"}'
    selector: pair
  inside:
    kind: object
    inside:
      kind: pair
      stopBy: end
    stopBy: end
```

For YAML, even simpler: `"$KEY: $VAL"` as a direct pattern works.

This means walk.rs may not be needed for JSON/YAML -- ast-grep can be the unified engine for source code + structured data. Only TOML falls back to serde walker (unless custom language support works).

## Language Design (current shape)

Tree selector with > descent, captures at every level:
```
(app1|libA)
> /main|support\/10\..*/
> ./**/package.json
> json.*.{
    name: $NAME,
    (devDependencies|dependencies): { $DEP: $VER },
    scripts: { /^(pre|post)?build/: $CMD },
    workspaces: [...$WS_GLOB],
  }
```

Key position accepts any pattern: literal, glob, regex, alternation, $CAPTURE.
Every match fans out. Colon separates key from value capture.
<<>> for embedded pattern blocks. Spread ... for arrays.
