## Key Decisions This Session
- DOM mental model: repos as elements, files as children, parsed tree as nested nodes
- Captures on steps (not separate fields), Object step for sibling capture
- Value regex with source field for splitting (scoped npm packages)
- Action emit array with parent linkage for grouped refs
- Batch DB writes to avoid n+1 (accumulate in memory, flush at threshold)
- Parallel parse via rayon, single-threaded DB write
- Pre-compile all globs/regexes at rule load time
- Normalization should eventually be per-rule via transform pipes
- URTSL is the authoring language, JSON RuleSet is the IR, compiled Rust is the target
