## Key Insights
- Cross-repo link rules have a scan-order dependency: target-side repos must be indexed before source-side repos trigger link resolution. Known gap -- a second link-rule pass after all repos are scanned would make order irrelevant.
- insta snapshot workflow: `cargo insta accept --all` or `find . -name "*.snap.new" | while read f; do mv "$f" "${f%.new}"; done`. INSTA_UPDATE=always should work but sometimes writes .snap.new instead.
- sccache can't cache proc-macros (linker involved). Handles lib crates well. Hit rate ~60-80% for typical workspace.
- cargo clean is almost never needed.
