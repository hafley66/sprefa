## Solution/Approach

### matches_v2 -> matches rename
Dropped the old `matches` table DDL (which referenced a `rules` table by id). Renamed `matches_v2` to `matches` across 7 source files + migrations. Updated index names and match_labels FK reference.

### ActionKind enum removal
- `EmitRef.kind` changed from `ActionKind` to `String`
- Deleted `ActionKind` enum and `to_kind_str()` impl from types.rs
- Removed `use sprefa_extract::kind` from types.rs
- emit.rs: `emit.kind.to_kind_str().to_string()` -> `emit.kind.clone()`
- Tests: replaced `ActionKind::X` with `kind::X.into()` or string literals
- Deleted `action_kind_to_kind_str` test (no enum to test)
- Replaced `reject_invalid_kind` test with `accept_arbitrary_kind_string`
- Updated 2 snapshots (schema, scoped packages sort order)

### insta snapshot issue
`cargo test` (workspace) and `cargo test -p sprefa_rules` produce different schemars JSON schema output (likely feature flag or compilation order). Fixed by running `INSTA_UPDATE=always cargo test` from workspace root.
