# Session: sprefa-index-cache-crate-split

## Goal
Split `crates/scan` into `crates/index` (pure Rust extraction) and `crates/cache` (sqlx flush), with `crates/scan` becoming a thin coordinator.

## Current State
Split is complete and all 71 tests pass. Last commit: `66447a9` (pre-split -- not yet committed).

## Problem/Context
`crates/scan` contained everything: file enumeration, parallel extraction, and all sqlx DB write logic. The architecture decision called for separating pure extraction from DB writes so that testing extraction requires no DB setup.

## Solution/Approach
- `crates/index`: `files.rs` (git ls-files + walkdir), `normalize.rs`, `extract.rs` (par_iter loop + `ExtractedFile` type). No sqlx, no tokio, no async.
- `crates/cache`: `flush.rs` (all sqlx bulk insert logic + 5 tests rewritten to call `flush()` directly). Depends on `sprefa_index` for `ExtractedFile` and `normalize`/`normalize2`.
- `crates/scan`: thin coordinator -- `Scanner` struct + `scan_repo` that calls `sprefa_index::extract` then `sprefa_cache::flush`.
- Snapshot files moved to `crates/index/src/snapshots/` with updated `sprefa_index__` prefix.
- Root `Cargo.toml` workspace members updated to include `crates/index` and `crates/cache`.
