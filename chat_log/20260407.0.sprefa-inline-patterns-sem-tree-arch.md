## Key Insights
- `SegmentCapture` in pattern.rs already does all the hard work for inline patterns. Implementation was just wiring it through the parser and walk engine.
- sem (Ataraxy-Labs/sem) is Rust/tree-sitter/SQLite -- same stack. Entity-level extraction + dependency graph. BUT its symbol resolution is just name-based tokenization (no scope, no module paths, no type inference). sprefa's oxc/syn extractors already have better module resolution for JS/TS and Rust.
- The pluggable architecture: TreeNode trait as the unifier, tag registry replacing the Tag enum, constraint walker decoupled from JSON. DOM/CSS analogy: unified tree = DOM, .sprf patterns = CSS selectors, tag registry = custom elements registry.
- ast-grep `inside` chaining maps naturally to .sprf block nesting. sprefa generates ast-grep rules, doesn't extend ast-grep.
- `scan()` tag is redundant -- `repo()`/`rev()` as bare steps already set scan annotations via collect_scan_annotations.
- `${VAR}` delimiter syntax needed for segment patterns when surrounding text contains `_` (e.g., `use${ENTITY}Query`).
