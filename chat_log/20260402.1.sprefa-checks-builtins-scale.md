- README builtin table updated

### New builtins
- `$.same_norm2`: matches A and B share the same norm2 (suffix-stripped normalization)
- Joins through strings table on norm2 column, same pattern as $.same_norm but on norm2

## Key Insights
- Variable unification in queries handles exact cross-repo string matching for free -- no special builtin needed for "same title" comparisons
- The check-from-copy-side pattern (check violations on the copy, not the source) correctly handles N copies where some are current and some are stale
- Cartesian product in discover_scan_targets (file_id join) produces ghost targets when multiple repo/rev pairs exist in one file -- handled gracefully by catching scan errors, proper fix needs group_id-aware discovery query
- `$.` prefix for builtins is a clean visual separator in .sprf files and avoids name collisions with user rules
