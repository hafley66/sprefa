# Session: sprefa-kill-matches-table-done

## Goal
Eliminate the `matches` table by migrating built-in extractors (JS, RS) to use per-rule tables, unifying the two parallel storage paths.

## Current State
Complete. Committed as 319537c. All 381 tests pass. matches table dropped.

## Problem/Context
Two parallel storage paths existed:
1. Built-in extractors (JS/RS) -> `refs + matches` (polymorphic kind column)
2. .sprf rule extractors -> per-rule `_data` tables (typed columns)

`matches` was a free-floating relation table doing the same job as per-rule tables but worse.

## Solution/Approach
7-step sequential implementation:
1. Added `builtin_rule_table_defs()` to rule_tables.rs (10 tables, single "value" column)
2. Registered builtins at startup in main.rs (no sprf_meta hashing)
3. Flipped JS/RS extractors: rule_name=kind constant, group=Some(counter++)
4. Rewrote all 10 query functions in watch/queries.rs + resolve.rs (JOIN matches -> JOIN {kind}_data)
5. Rewrote watcher.rs load_decl_refs (UNION ALL over per-rule tables) and replace_file_refs (per-rule INSERT/DELETE)
6. Fixed search_refs (deleted broken repo_refs UNION arm) and flush.rs (per-rule cascade deletes)
7. Dropped matches DDL, deleted flush_repo_meta, resolve_links stub, fixed test helpers, accepted 19 snapshots
