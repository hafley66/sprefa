# Session: sprefa-line-matcher-rename

## Goal
Rename `re()` tag to `line()`, make segment capture the default pattern mode, `re:` prefix for raw regex. Rename `ValuePattern` to `LineMatcher` enum.

## Current State
All changes landed and compiling. Two new tests pass (`lower_line_segments`, `lower_line_regex`). Pre-existing snapshot and discovery_e2e failures unrelated to this work.

## What Changed (code)
- `crates/rules/src/types.rs` -- `ValuePattern` struct replaced with `LineMatcher` enum: `Segments { source, pattern }` and `Regex { source, pattern, full_match }`. Serde tagged by `mode`.
- `crates/rules/src/emit.rs` -- `apply_value_pattern` -> `apply_line_matcher`. Segments variant calls `parse_segment_pattern` + `match_segments_pub`. Regex variant keeps existing logic.
- `crates/sprf/src/_0_ast.rs` -- `Tag::Re` -> `Tag::Line`, parses `"line"`.
- `crates/sprf/src/_3_lower.rs` -- `Tag::Line` handler detects `re:` prefix for regex mode, defaults to segment capture. All `value_pattern` locals -> `line_matcher`. Two new tests.
- `crates/rules/src/extractor.rs` -- `value_pattern` field -> `line_matcher`, type `ValuePattern` -> `LineMatcher`.
- `crates/rules/tests/rules_test.rs` -- test constructors updated to `LineMatcher::Regex { ... }`.
- `crates/sprf-lsp/src/main.rs` -- completion entry updated.
- `editors/vscode/syntaxes/sprf.tmLanguage.json` -- `re-slot` -> `line-slot`.
- `EXAMPLES.sprf` -- all `re(` -> `line(`.
- `crates/sprf/src/_1_parse.rs` -- doc comment updated.
