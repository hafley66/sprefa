## Key Insights
- Scan is a single-pass pull pipeline: config -> skip_set -> file walk -> parallel extraction (rayon) -> flush (one SQLite tx: strings -> files -> branch_files -> refs -> matches) -> resolve import targets
- Watch is a push-driven event loop with three async boundaries: OS events ~> debounce(100ms) ~> classify/correlate ~> plan channel(32) ~> apply. Self-terminates via quiescence (re-extraction after rewrite produces no diff)
- parent_key_string_id is the existing intra-file link primitive (dep_version -> dep_name, within same file, same rule)
- target_file_id is the existing cross-file link primitive (import_path -> resolved file), populated in post-flush resolve pass using oxc_resolver
- Link rules would generalize these: connect import_path to export_name, dep_name to repo that provides the package, etc. Cross-file, cross-kind
- The pause AtomicBool on the watcher gates event processing during external operations (git checkout)
- Binary hash (SPREFA_GIT_HASH) invalidates the entire skip set when extractors change -- coarse but correct
- Watch mode does wt branch management: full-replace branch_files on each scan, additive for committed branches
- Watch classify_batch correlates moves by content_hash (delete+create with same hash = move)
- Atomic save detection: same-path delete+create with different hash = content change
- diff_refs uses span proximity (64 byte threshold) to match old/new declarations for rename detection
