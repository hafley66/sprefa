# Session: sprf-meta-implementation

## Goal
Implement `sprf_meta` - a rule change detection system that tracks schema and extract hashes for each rule to enable incremental re-extraction. This allows sprefa to skip unchanged rules, re-extract only modified rules, and automatically handle schema changes.

## Current State
**Nearly complete** - Core implementation done, E2E tests need compilation fixes and verification.

## Problem/Context
- sprefa was doing full re-extraction on every run, wasting time on unchanged rules
- Need to detect: (1) schema changes → DROP+CREATE table, (2) extract logic changes → DELETE cached rows, (3) no changes → skip entirely
- Hash computation must include transitive dependencies (if rule A uses rule B's output, A's hash depends on B's hash)
- Should fail gracefully if sprf_meta table is missing/corrupt

## Solution/Approach
1. **Cleanup first**: Remove legacy `matches`/`match_labels`/`flush.rs` code
2. **Hash computation**: Topological sort of rules, compute schema_hash + extract_hash with transitive propagation
3. **Change detection**: Compare stored hashes vs computed hashes, return `RuleChangeKind` enum
4. **Table creation**: `create_rule_tables()` accepts optional hashes, handles SchemaChanged/ExtractChanged/Unchanged
5. **Integration**: Wire into CLI's `build_scanner()` function
6. **E2E tests**: Verify caching works and new rule files trigger re-scans
