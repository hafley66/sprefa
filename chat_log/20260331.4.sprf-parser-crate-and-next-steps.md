# Session: sprf-parser-crate-and-next-steps

## Goal
Build the crates/sprf/ parser crate that takes .sprf DSL text and lowers it into the existing Rule/SelectStep type system. Then wire it into the CLI.

## Current State
Committed (9bc0f7c). The sprf crate is complete with 30 tests passing, CLI wired, full integration tests running against this repo's own Cargo.toml. The user said "now we stop punting on the next steps" -- meaning output slot syntax for capture-to-match mapping is the next work item.

## Problem/Context
Previous session made SelectStep recursive (Object with entries, Array with item sub-chains, KeyMatcher enum). This session built the parser that targets those types. The .sprf DSL uses CSS function notation: tag(body) with paren-counted delimiters, > separators, ; terminators, # comments.

## Solution/Approach
Four-file parser crate:
- _0_ast.rs: SelectorChain, Slot, Tag (parse tree)
- _1_parse.rs: hand-rolled parser (paren counting, bare glob detection, tag dispatch)
- _2_pattern.rs: json() body -> Vec<SelectStep> tree (recursive object/array/capture/wildcard)
- _3_lower.rs: SelectorChain -> Rule (bare inference N=3, tag dispatch)
- lib.rs: parse_sprf() + integration tests against real repo files

winnow is a dep but the parser ended up hand-rolled (cleaner for paren counting). winnow unused imports were cleaned up.

## Key Insights
- Bare glob inference: exactly 3 bare globs = repo > branch > fs. Anything else errors. A lone bare glob before a tagged slot like json() is NOT allowed -- must use fs() tag.
- re: key prefix in json() body passes through as KeyMatcher::Glob("re:...") which the walk engine's pipe_glob_matches already handles
- ** as object key lowers to SelectStep::Any (recursive descent)
- create_matches is currently empty on .sprf-lowered rules -- the output slot syntax is the critical next piece
- The user explicitly listed punted items last session: output slots, parent capture on MatchDef, link rules, query subcommand. Output slots are now un-punted.

## Open Questions

- re() source capture convention: currently defaults to empty string, needs a way to specify which capture the regex runs against
- Should winnow dep be removed since parser is hand-rolled?
