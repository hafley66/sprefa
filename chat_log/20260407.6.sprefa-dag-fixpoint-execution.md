# Session: sprefa-dag-fixpoint-execution

## Goal
Implement DAG-guided fixpoint execution (#2b from BOARD.md). Replace "run all rules every round" discovery with DAG-pruned level-walk where cross-rule dependencies control discovery ordering.

## Current State
Implementation complete, all tests passing (240 pass, 2 pre-existing extraction_e2e failures unchanged). Code is uncommitted.

## What Changed

### Phase 1: graph.rs (new)
- `crates/rules/src/graph.rs`: `DepEdge`, `RuleGraph`, `build_rule_graph()` 
- Kahn's algorithm topo-sort into levels
- Cycle detection with DFS path reporting
- 7 unit tests: no-edges, linear chain, diamond, cycles, unknown producer, independent+chain

### Phase 2: Dep edge extraction in lowering
- `crates/sprf/src/_3_lower.rs`: `lower_program` now returns `(RuleSet, Vec<DepEdge>)`
- New `collect_dep_edges()` recurses through `RuleBody::Ref` nodes
- 2 new tests: cross_ref_produces_dep_edge, no_cross_refs_no_edges

### Phase 3: API signature propagation
- `crates/sprf/src/lib.rs`: `parse_sprf` and `load_sprf` return `(RuleSet, Vec<DepEdge>)`
- `crates/cli/src/main.rs`: `load_ruleset` returns `(RuleSet, Vec<DepEdge>)`, JSON/YAML paths return empty edges
- All callers updated (integration tests, discovery_e2e, extraction_e2e, watch tests)

### Phase 4: Scanner + discovery loop
- `crates/scan/src/scanner.rs`: `scan_pairs` field renamed to `scan_pair_levels: Vec<Vec<ScanPair>>`
- `crates/cli/src/main.rs` `build_scanner`: builds RuleGraph, groups scan_pairs by level
- Discovery loop rewritten as nested: outer fixpoint > level walk > per-level mini-fixpoint
- Tracing logs include level index and DAG stats

## Key Insights
- DAG controls discovery ordering, not per-rule extraction. Extraction is file-based (all rules run on every file). The DAG determines which scan_pairs to query when.
- CrossRef was fully parsed in AST but discarded during lowering. Now `collect_dep_edges` preserves it as DepEdge structs.
- Outer fixpoint loop still needed: higher-level scans can produce rows for lower-level rules (all rules extract from every file).
- For rulesets with no cross-refs, behavior is identical to before (single level with all scan_pairs).
