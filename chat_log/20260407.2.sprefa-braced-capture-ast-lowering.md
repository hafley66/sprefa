# Session: sprefa-braced-capture-ast-lowering

## Goal
Implement `${VAR}` braced delimiter syntax for segment capture patterns, then lower it into ast-grep rules for sub-identifier capture with precise byte spans.

## Current State
All work landed and committed as `1f250c2`. 35 pattern+ast tests pass, 12 lowering tests pass.

## What Changed (code)

### ${VAR} parse support (pattern.rs)
- `parse_segment_pattern` extended: after `$` or `$$$`, if next char is `{`, reads until `}` for name
- `capture_offsets_in_value` added: walks segments sequentially to compute byte offsets of each capture within matched text
- 7 new tests (3 parse, 2 match, 2 offset)

### ast-grep lowering (_3_lower.rs)
- `rewrite_ast_braced_captures(body)` scans ast pattern for identifier tokens containing `${...}`
- Each token replaced with synthetic metavar `$SPREFA0`, `$SPREFA1`, etc.
- Generates regex constraint for ast-grep filtering (e.g. `^use.+Query$`)
- Stores original token as segment pattern for post-match extraction
- Helper: `segments_to_constraint_regex` converts Segment vec to anchored regex
- Helper: `is_ast_ident_char`, `peek_brace` for token scanning
- `Tag::Ast` arm updated to call rewrite before constructing AstSelector
- 4 new tests (no-op passthrough, single capture, multiple captures, full lower)

### AstSelector post-processing (ast.rs, types.rs)
- `segment_captures: Option<BTreeMap<String, String>>` field added to AstSelector
- `collect_matches` extended: after ast-grep match, applies segment pattern to synthetic metavar text
- Precise spans: `metavar_start + local_offset` from `capture_offsets_in_value`
- 2 new tests (extraction with span assertion, non-matching filter)

### Also in this commit (from prior session)
- `re()` tag renamed to `line()`, `ValuePattern` -> `LineMatcher` enum
- `apply_value_pattern` -> `apply_line_matcher` with Segments/Regex variants
- tmLanguage, LSP completions, EXAMPLES.sprf updated

## Key Insights
- `$_` prefix in ast-grep means wildcard/anonymous -- synthetic metavars must avoid underscore prefix (use `$SPREFA0` not `$_SPREFA_0`)
- ast-grep constraints regex is filter-only (no group extraction) -- segment matching does the actual capture extraction as a post-processing step
- Token scanning for `${...}` in ast patterns: a token is a maximal run of ident chars + `${...}` sequences; `$` without `{` is treated as regular token char (preserves ast-grep `$NAME`/`$$$NAME` metavars)
- Span precision: `capture_offsets_in_value` walks segments sequentially, summing literal/capture lengths -- breaks on Wild/MultiWild (ambiguous without re-matching)
