## Goal
Add VS Code syntax highlighting and LSP for .sprf files. Update README with .sprf DSL docs.

## What was built
1. **TextMate grammar** (`editors/vscode/syntaxes/sprf.tmLanguage.json`) -- syntax highlighting for .sprf files in VS Code
2. **VS Code extension** (`editors/vscode/`) -- language config, bracket matching, comment toggling, LSP client
3. **sprf-lsp** (`crates/sprf-lsp/`) -- tower-lsp language server with:
   - Scoped capture completions (per-rule, not global)
   - Match kind suggestions in match() and link() positions
   - Link predicate completions
   - Tag name completions after >
   - Push diagnostics: parse errors + validation warnings (unmatched captures, unknown link kinds)
4. **Watcher filter** -- events from target/, node_modules/, .git/ etc filtered at notify callback level
5. **CLI reset command** -- `sprefa reset` drops and recreates the index DB
6. **README .sprf section** -- full grammar docs, examples, destructuring table
7. **.cargo/config.toml** -- nightly parallel frontend (-Z threads=8)
8. **Cargo.toml profile** -- opt-level=2 for deps, 0 for workspace crates

## Known Issues
- TextMate regex-key pattern: fixed `]` inside regex char classes breaking highlighting, now uses lazy match to `: ` separator
- re-slot and ast-slot: use lookahead `(?=\s*[>;#])` for paren-safe end detection
- LSP diagnostic positions: parse errors use heuristic (last unterminated statement), not exact byte offsets from parser
- sccache hit rate was 4.67% due to profile/toolchain changes triggering full recompile

## Deep Discussion: Rust compile times and Zig
Extended discussion about Rust compile times, dependency bloat, and Zig as alternative:
- Workspace cold build takes minutes due to ~300 transitive deps
- tokio/axum/sqlx/tower-lsp each pull massive dep trees
- Zig: comptime instead of monomorphization, @cImport for C libs (zero bindings), arena allocators as first-class pattern
- Realistic dep reduction: sqlx->rusqlite, tower-lsp->lsp-server, axum->tiny-http, git2->shell out to git
- User strongly interested in Zig for future work, wants perf test suite before any port
- D language discussed as well-designed but ecosystem-dead
