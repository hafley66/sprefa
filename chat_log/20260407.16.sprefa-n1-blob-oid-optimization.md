# Session: sprefa-n1-blob-oid-optimization

## Goal
Eliminate N+1 queries, unnecessary filesystem I/O, and fix the dead skip mechanism in the sprefa scan pipeline. Four compounding problems identified and planned.

## Current State
Plan written and approved at `/Users/chrishafley/.claude/plans/linked-sparking-tarjan.md`. No implementation started. Context budget exhausted before coding began.

## Problem/Context
Four compounding inefficiencies discovered:
1. **N+1 file inserts**: `flush_batch` calls `ensure_file` per-file (INSERT + SELECT each) + per-file `INSERT OR IGNORE INTO rev_files`. 1000 files = 3000 SQL roundtrips.
2. **Unconditional mmap+hash**: `extract_from_list` opens/mmaps/xxh3-hashes every file BEFORE skip check. 50k file repo, 49k unchanged = 49k wasted opens.
3. **Dead skip mechanism**: `ensure_file` never writes `scanner_hash` to files table. `load_scan_context` filters by `scanner_hash = ?`, matches zero rows. Skip set is ALWAYS EMPTY. Nothing is ever skipped.
4. **All-or-nothing staleness**: `has_stale_scanner_hash` returns bool. One stale file forces full rescan of entire repo.

Key insight: `git2::TreeEntry::id()` returns blob OIDs (SHA-1) for every tracked file WITHOUT reading content. Use as `content_hash` instead of xxh3. Skip check becomes pure in-memory lookup.

## Solution/Approach
Three phases in the plan file:

### Phase 1: Fix Dead Skip + Bulk File Inserts
- 1a: Add `scanner_hash` param to `flush_batch`, actually write it to files table
- 1b: Replace per-file `ensure_file` loop with bulk INSERT...RETURNING pattern (chunked by FILE_CHUNK=2000). Bulk rev_files INSERT. Delete `ensure_file` method.

### Phase 2: Blob OIDs from Git, Defer mmap
- 2a: `git2_list_files` returns `Vec<(PathBuf, String)>` with blob OIDs. Same for `diff_files`, `list_blobs_at_rev`.
- 2b: `extract_from_list` accepts pre-computed hashes, skip check BEFORE mmap. Zero I/O for unchanged files.
- 2c: Thread blob OIDs through scanner pipeline.

### Phase 3: Incremental Staleness
- 3a: Replace `has_stale_scanner_hash` (bool) with `stale_file_paths` (Vec<String>)
- 3b: In `scan_diff`, merge stale files into diff.changed instead of bail!("full rescan")

## Key Insights
- scanner_hash is NEVER written by ensure_file, making the entire skip_set mechanism dead code
- git2 tree walk exposes entry.id() (blob OID) without reading blob content -- already used for rename detection at files.rs:69
- RETURNING clause works with multi-row INSERT in SQLite 3.35+ and is used extensively elsewhere in the codebase
- Hash transition is clean: since scanner_hash was never written, skip_set is empty, first scan after change populates blob OIDs + scanner_hash
- Non-git fallback (walkdir_files) keeps mmap+xxh3, rare path
- flush.rs delete/rename operations are already well-designed (temp table pattern), only file inserts are N+1
