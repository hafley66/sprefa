## Problem/Context
User has multiple microservices with openapi.json files in monorepo packages that are manually copied to source-of-truth repo. Names don't match exactly (user-service vs user), versions drift. Need to:
- Detect version mismatches between monorepo packages and SOT
- Handle name normalization (strip -service, -api, -svc suffixes)
- Flag orphan packages (no matching SOT entry)
- Record violations without halting scan
- Query violations later via CLI

## Solution/Approach
**Raw SQL check blocks** — simplest design:

```sprf
check(openapi_drift) {
    SELECT m.name as package, m.version as actual, s.version as expected
    FROM mono_openapi_data m, sot_openapi_data s
    WHERE strip_suffixes(m.name, '-service') = s.name
      AND m.version != s.version
};
```

**Semantics:** Run SQL after scan, each row returned → insert into `invariant_violations` table.

**Key design decisions:**
- No quotes around SQL (brace-delimited)
- TextMate grammar embeds `source.sql` inside check blocks
- UDFs registered on SQLite connection open
- Auto-run after every scan, `sprefa check` queries stored violations
- Violations persist in DB for history tracking
