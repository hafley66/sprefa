## Key Insights
- RsUse refs per repo are small (hundreds to low thousands), so fetch-all + filter in Rust is fine
- The rewriter (rewrite_use_path, reconvert_prefix) already handled all prefix styles correctly; the gap was purely at the discovery layer
- Self-healing loop: watcher's own rewrites trigger ContentChange events in the next debounce batch, which re-extracts and syncs DB automatically
- DB staleness between batches is ~100ms (debounce window), acceptable for the model
- For git-op bursts: threshold (>30 events) + skip classify/plan/rewrite + cooldown + full re-scan is the proposed approach
