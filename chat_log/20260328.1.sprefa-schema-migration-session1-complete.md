## Solution/Approach

### Bug fix: flush.rs tuple field swap
Root cause of live watcher test timeouts: in `crates/cache/src/flush.rs:240`, the match_rows tuple was built as `(ref_id, kind, rule_name)` but destructured as `(ref_id, rule_name, kind)` to match the SQL column order. Every matches_v2 row had kind and rule_name swapped, so `load_decl_refs` (which filters by `m.kind IN ('export_name', ...)`) found nothing, and `diff_refs` produced `Added` instead of `Rename` changes. One-line fix: swap the tuple construction order.

### Kind constants
Added `pub mod kind` in `crates/extract/src/lib.rs` with 10 `pub const &str` values. Replaced ~200 scattered string literals across 10 files. SQL string literals in queries.rs left as-is (inside SQL text). schema/types.rs RefKind methods left as-is (schema doesn't depend on extract; RefKind is demoted).

### N+1 UPDATE batch
Replaced per-row `UPDATE refs SET target_file_id = ? WHERE id = ?` loop in `crates/cache/src/resolve.rs` with batched `CASE WHEN id = ? THEN ? END WHERE id IN (...)` -- one statement per chunk of 500.
