# Session: sprefa-sprf-tree-selector-language

## Goal
Design the .sprf language. NOT datalog. NOT SQL-like. A tree selector language with captures at every level, driven by glob/regex/ast-grep patterns. `>` for descent.

## Current State
Language shape is converging. No code written. The datalog/SQL direction is shelved. The language is a tree path selector -- repo > branch > filesystem > content -- with pattern matchers and $CAPTURES at every position.

## Core Design

Everything is a tree. One selector walks all levels with `>` descent:

```
(app1|libA)                                    ← repo pattern
> /main|support\/10\..*/                       ← branch pattern (regex)
> ./**/package.json                            ← file glob
> json.*.{                                     ← content: iterate top-level values
    name: $NAME,                               ← capture leaf
    (devDependencies|dependencies): {          ← alternation on key
      $DEP: $VER                               ← capture key + value per entry
    },
    scripts: { /^(pre|post)?build/: $CMD },    ← regex key match
    workspaces: [...$WS_GLOB],                 ← spread array elements
  }
```

### Pattern vocabulary (uniform at ALL tree levels)
- `"literal"` -- exact string
- `(a|b|c)` -- alternation
- `/regex/flags` -- regex
- `glob*pattern` -- glob
- `$CAP` -- wildcard + capture
- `*` -- wildcard, no capture

### Content tree (JSON/YAML/TOML)
- `.key` -- literal descent
- `.$CAP` -- wildcard key, capture name
- `.*` -- wildcard key, no capture
- `{ key: $VAL, ... }` -- destructure object, multiple captures
- `{ $KEY: $VAL }` -- iterate object, capture both key and value
- `[...$ITEMS]` -- spread array, capture each element
- `key: $CAP` -- colon separates key from value capture
- Nested `{}` for deeper objects -- no shorthand, fully explicit

### Source code files (ast-grep/regex)
- `ast <<import { $Name } from '$Source'>>` -- ast-grep pattern in `<<>>` delimiters
- `re <<image:\s*(.+)>> as Value` -- regex with positional binding

Two selectors share a variable → join on that binding's value.

## Key Decisions
- `.sprf` file extension
- `>` for tree level descent (back on the table, CSS child combinator)
- `<<>>` for embedded pattern blocks
- Commas delimit within destructure blocks
- Every pattern fans out -- regex matching 3 keys produces 3 binding sets
- No SQL. No datalog. Pure tree selector with captures.
- No shorthands first pass -- every nesting level explicit
- Spread `...` for arrays (JS/Rust convention)
- Concrete matched key stored in DB alongside captures
- Globs bare when no spaces, quoted when spaces

## Shelved
- Datalog syntax for link/query rules (may return in different form)
- `from/where/select` BigQuery pipe syntax
- Type annotations on captures
- jaq/jq integration (walk.rs covers 90% of cases)

## Prior Art Studied
- tree-sitter queries (closest: S-expression patterns with @captures)
- JSONPath, JMESPath, jq (extraction-oriented, no capture holes)
- XPath (original tree query, verbose but powerful)
- gron (flatten JSON to path=value -- conceptual inverse)
- CSS selectors (descent combinators, pseudo-classes)
