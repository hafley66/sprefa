# Session: Multi-Rev Scanning + IS_REPO/IS_REV Discovery

## Goal

Enable scanning file trees at arbitrary git revisions (tags, branches) without checkout, driven by two mechanisms:
1. **Static** (`scan_tags` config): scan all tags matching a glob pattern
2. **Dynamic** (match annotations): extraction rules mark captures as `IS_REPO` / `IS_REV`, scanner collects pairs and chases them

## Problem

The scanner reads files from disk via mmap. It can only index HEAD. Tags are stored as `repo_refs` (string metadata), but their file trees are never walked. So:
- "deploy chart references v2.3.1" connects to "repo X has tag v2.3.1" via `same_norm` -- works
- "what are the npm deps in repo X at v2.3.1?" -- fails, `package-lock.json` at that commit was never read

## Architecture

### Current flow (single-rev, filesystem-only)

```
sprefa.toml             .sprf rules
    |                       |
    v                       v
cmd_scan()           RuleExtractor
    |                       |
    v                       v
for repo in repos:    extractors[]
  for branch:              |
    scan_repo()            |
      |                    |
      v                    v
  list_files()      extract_from_list()
  (git2 HEAD tree)  (mmap from disk)
      |                    |
      v                    v
  flush() ---------> refs, matches, branch_files
      |
      v
  resolve_links()
```

### New flow (multi-rev, blob reader + discovery)

```
sprefa.toml             .sprf rules
    |                       |
    |  scan_tags: ["v*"]    |  match($REPO, image_repo, IS_REPO)
    |                       |  match($TAG, image_tag, IS_REV)
    v                       v
cmd_scan()           RuleExtractor
    |                       |
    |--- TIER 1 -----------|
    |                       |
    v                       v
 for repo:            extractors[]
   for branch:             |
     scan_repo()           |  <-- existing, filesystem
       |                   |
   for tag in scan_tags:   |
     scan_rev()            |  <-- NEW, git2 blobs
       |                   |
       v                   v
   list_blobs_at_rev()  extract_from_blobs()
   (git2 tree at rev)   (blob bytes, no disk)
       |                   |
       v                   v
   flush() ----------> refs, matches, match_labels(scan=repo/rev)
       |
       v
   resolve_links() (2nd pass cross-repo)
       |
    |--- TIER 2 (discovery loop) ---|
    |                               |
    v                               |
  query match_labels                |
  for (repo, rev) pairs             |
    |                               |
    v                               |
  filter: repo in config?           |
  filter: rev already scanned?      |
    |                               |
    v                               |
  scan_rev(repo, rev)               |
    |                               |
    v                               |
  resolve_links()                   |
    |                               |
    +--- loop until no new targets -+
```

### Match annotation data flow

```
.sprf file                         database
----------                         --------

match($REPO, image_repo, IS_REPO)
  |
  v (parse)
Slot::Match { scan: Some(Repo) }
  |
  v (lower)
MatchDef { scan: Some("repo") }
  |
  v (emit)
RawRef { scan: Some("repo") }
  |
  v (flush)
matches row + match_labels row
  key="scan", value="repo"
```

### Discovery SQL join pattern

```
match_labels                    refs                    strings
(scan="rev")                    (file_id, string_id)    (value)
     |                               |                      |
     v                               v                      v
  rev_match ---- ref_id ----> rev_ref ---- string_id --> rev_string = "v2.3.1"
                                 |
                    parent_key_string_id
                                 |
                                 v
                           repo_ref ---- string_id --> repo_string = "org/backend"
                                 ^
                                 |
                          repo_match ---- match_labels (scan="repo")
```

The `parent_key` linkage on refs connects the tag match to its parent repo match. The discovery query follows this join to produce `(repo_name, rev_string)` pairs.

### Git blob reader vs filesystem reader

```
FILESYSTEM (existing)                GIT BLOB (new)
-----------------------              ----------------------
list_files(repo_path)                list_blobs_at_rev(repo_path, "v2.3.1")
  git2: HEAD -> tree                   git2: revparse("v2.3.1") -> tree
  tree.walk -> rel_paths               tree.walk -> (rel_path, blob_oid)
                                         blob.content() -> bytes

extract_from_list(paths)             extract_from_blobs(blobs)
  fs::File::open(abs_path)            bytes already in memory
  mmap -> &[u8]                        &blob_bytes
  xxh3_128(&mmap)                      xxh3_128(&blob_bytes)
  extractor.extract(&mmap, ...)        extractor.extract(&blob_bytes, ...)
  -> ExtractedFile                     -> ExtractedFile (same type)
```

Both paths produce `Vec<ExtractedFile>`. The flush pipeline is identical downstream.

## Implementation order

1. `IS_REPO`/`IS_REV` annotations: AST -> parser -> lower -> MatchDef -> RawRef -> emit -> flush
2. `list_blobs_at_rev()` in `crates/index/src/files.rs`
3. `extract_from_blobs()` + `extract_rev()` in `crates/index/src/extract.rs`
4. `scan_rev()` on Scanner
5. `scan_tags` config field on RepoConfig
6. Tag scanning loop in `cmd_scan`
7. Discovery loop in `cmd_scan`

## Key decisions

- No schema migration: `match_labels` table exists unused, `branch_files` accepts arbitrary branch strings
- No `discover` config block: discovery driven by `IS_REPO`/`IS_REV` in .sprf rules
- `parent` field on MatchDef pairs repo+rev matches (existing mechanism)
- Skip `resolve_import_targets` for rev scans (historical snapshots)
- Fixed-point capped at 10 iterations
