# Session: store-trait-perrule-tables

## Goal
Execute the sprefa zoom-level-2 implementation roadmap: Store trait, per-rule tables, query engine removal.

## Current State
Phase A done, Phase B partial. Store trait defined as complete data boundary. SqliteStore implements flush_batch with string interning, ref insertion, per-rule table rows. RuleTableDef generates DDL for {name}_data tables, {name} value views, {name}_refs provenance views. 60 core tests pass.

## Problem/Context
sprefa's storage layer had raw sqlx scattered across 6+ cache modules. The schema used EAV shredding (individual match rows reconstructed via group_id self-joins, 3-hop value resolution). ~700 LOC query compiler reimplemented SQLite. User mandated: ALL data storage goes through Store trait boundary ("we could output to json instead of sqlite one day").

## Solution/Approach
- Store trait in cache/store.rs is the complete data boundary (not just discovery)
- SqliteStore in cache/sqlite_store.rs is the only place sqlx lives for writes
- Per-rule tables: one row per extraction event, dual ref/str columns per capture
- Query engine (query.rs, CTE compiler, builtins) gets deleted entirely -- raw SQL blocks replace it
- Links removed as a language concept
