## What was built

### Rust path rewriter (`crates/watch/src/rs_path.rs`)
- `file_to_mod_path()` converts file paths to module paths (src/foo/bar.rs -> crate::foo::bar)
- `RsPathRewriter` implements `PathRewriter` trait for Rust use statement rewriting
- Handles crate::, self::, super:: prefixes; preserves prefix style when possible
- 11 tests covering all cases

### Rust use-path queries (`crates/watch/src/queries.rs`)
- `rs_uses_with_prefix()` -- find RsUse refs by module path prefix (file moves)
- `rs_uses_ending_with()` -- find RsUse refs by module path + trailing name (decl renames)

### Extended plan_rewrites (`crates/watch/src/plan.rs`)
- `plan_file_move()` now queries and rewrites RsUse refs when .rs files move
- `plan_decl_rename()` handles RsDeclare renames (was ExportName-only)

### FS watcher (`crates/watch/src/watcher.rs`)
- notify v8, debounce 100ms, content-hash move detection
- Classifies events into FsChange + DeclChange, feeds plan_rewrites

### CLI commands (`crates/cli/src/main.rs`)
- `sprefa watch [--repo]` -- watch only, no HTTP, no scan
- `sprefa daemon [--repo] [--no-scan]` -- scan + watch + serve all-in-one
- `--json` global flag for structured JSON logging

### Structured logging

- Structured fields: phase, repo, change_count, edit_count, file, reason, error
- Server scan lock acquire/release/timeout events logged

### Docs

## Key Insights
- Rust module path rewriting is fundamentally different from JS: use paths are strings not file refs, so queries use string prefix matching rather than target_file_id joins
- RsUse refs don't have target_file_id set (resolve only handles ImportPath), so Rust file move detection queries by string value prefix
- super:: prefix preservation requires resolving to absolute, substituting, then reconverting -- falls back to crate:: when the new path escapes the relative scope
- The watcher queries the index but doesn't update it after applying edits, meaning the index goes stale after rewrites. Re-scan needed periodically.
- oxc_resolver returns canonicalized paths; must canonicalize root_path too or lookups fail on macOS (/var -> /private/var)
- Pre-existing snapshot diff in sprefa_rules (JSON key ordering) is unrelated to this work
