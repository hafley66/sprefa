# Session: sprefa-multi-rev-scan-is-repo-is-rev (continued)

## Goal
Implement multi-rev scanning: index file trees at arbitrary git revisions (tags, branches, SHAs) without checkout, driven by static config and dynamic discovery via IS_REPO/IS_REV match annotations in .sprf rules.

## Current State
All 6 implementation tasks from the original plan are COMPLETE and tested:

3. Blob-based extraction (extract_from_blobs, extract_rev)
4. scan_rev() on Scanner

## Key Insights
- git2 revparse_single resolves tags, branches, SHAs identically -- the branch/tag distinction is a config-level fiction
- Discovery SQL parent_key join was wrong: walk engine assigns each sibling its own parent_key (the immediate key name), not a shared parent. file_id join is correct for same-file correlation.
- is_rev_scanned dedup works because flush() inserts into repo_branches with the rev string as branch key
- Cartesian product in multi-service files is harmless (scan_rev on nonexistent tag fails gracefully) but wasteful
- The null column cost of unification is minimal (is_working_tree DEFAULT 0, is_semver DEFAULT 0) and the code path reduction is significant
