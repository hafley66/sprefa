## Key Insights
- "Autism mode" subagent prompts: 300-400 line briefings with exact function signatures, every caller to update, every test with rationale. Agents execute mechanically, no judgment calls needed. All 5 landed clean.
- ATTACH DATABASE ':memory:' doesn't work with connection pools (per-connection state). Steered to table name prefixing instead.
- Monomorphization: added `is_chain: bool` to RuleBody::Block to distinguish `>` pipelines (sequential) from `{ }` brace blocks (branch-capable)
- $current* uses camelCase intentionally. AST capture collector only picks up SCREAMING_CASE, so they're invisible to create_matches for free. Pattern matcher sees them via parse_segment_pattern which is case-insensitive on captures.
- Pre-bound captures in match_segments_inner act as constraints: if name already in captures map, strip-prefix instead of trying candidates.
- Context forks (^repo syntax from board) turned out to be unnecessary -- siblings in blocks solve the same problem.
