# Session: rule-syntax-dag-fixpoint-design

## Goal
Design the next syntax evolution for .sprf rules and the execution model for cross-rule dependencies.

## Current State
Design complete, written to BOARD.md items #2 and #2b. No code changes yet.

## Key Decisions

### Syntax: `rule(name) { body };`
- Name in parens, body in braces, semicolons kept
- Captures inferred from body `$VAR`s, no head arg list
- Annotations stay inline: `repo($REPO)`, `rev($TAG)` at capture site
- Cross-rule references: `rulename(col: $REPO, col: $PIN)` -- named column binding
- Nesting for scoping, sibling order for sequencing (left-to-right capture flow)

### Execution: DAG-guided fixpoint
- Fixpoint loop stays as the runtime (correctness guarantee, supports future Prolog-style features)
- DAG is a compile-time acceleration structure that tells the fixpoint what to skip
- Cross-rule refs parsed into dependency edges, topo-sorted into levels
- Each round: skip rules whose inputs haven't changed (dirty propagation)
- Watermarking via SQLite rowid high-water marks per dependency edge
- Self-referential rules (recursive discovery) run as mini-fixpoint within their DAG node
- Independent rules at same level run in parallel

### Why fixpoint over pure topo-sort
- Pure topo-sort locks out recursive discovery and future bidirectional query features
- Fixpoint handles any pattern including ones not yet designed
- DAG just prunes redundant passes -- remove it and system still works, just slower
- Cross-ref syntax serves dual purpose: dependency edge in extraction context, table scan in query context

## What we verified
- `pattern.rs` already has full `$VAR`, `$$$VAR`, `$_`, `$$$_` segment capture support
- `_1_parse.rs` already produces recursive tree (RuleBody::Block)
- `_3_lower.rs` flattens tree but captures flow through pattern strings (not the `capture: Option<String>` field)
- `extractor.rs` runtime already handles multi-capture via `matches_with_captures()` on git/file selectors
- The `capture: None` in lowering is fine -- segment pattern captures flow through a parallel path

## Prior art considered
- dbt `ref()` model -- explicit cross-model references, topo-sorted
- Terraform resource references -- attribute-level deps, plan then apply
- Datalog (Souffle, Differential Dataflow) -- fixpoint with stratification
- CSS selectors -- top-down only, no parent climbing, `:has()` is closest
- ast-grep `$$$` -- greedy only, anchor-based disambiguation
