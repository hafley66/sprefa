# Session: sprefa-ast-grep-rules-disk

## Goal
Wire ast-grep as a real extraction backend in sprefa's RuleExtractor, add production rules that use it, and fix Rust build artifact disk bloat.

## Current State
All work committed. Workspace tests green. sccache + cargo-sweep + cargo-cache installed and configured globally.

Recent commits:
- `7fa85e7` rules: package_name extraction, js-env-var-refs ast rule, dep/env link rules
- `8ffaf6e` ast-grep extraction backend: pattern, inline rule, and rule_file modes

## Problem/Context
- RuleExtractor previously only handled structured data (JSON/YAML/TOML via serde_json::Value). Source code files had no extraction path via the rule DSL.
- `AstSelector` existed in types.rs but was inert -- rules with only ast steps were filtered out at compile time.
- Laptop was hitting 0 bytes disk free from Rust target/ accumulation across projects.

## Solution/Approach

### ast-grep backend
- `AstSelector` expanded to three modes: `pattern` (simple string), `rule` (inline JSON object mirroring ast-grep's rule: field), `rule_file` (path to .yml relative to rules file)
- New `crates/rules/src/ast.rs`: `ast_match()` dispatches to `Mode::Pattern`, `Mode::Rule`, or `Mode::Rule(from yaml file)`. Uses `DeserializeEnv::new(lang)` + `SerializableRuleCore::get_matcher(env)` from ast-grep-config.
- `RuleExtractor` gained `config_dir: Option<PathBuf>` for resolving rule_file paths. `from_json`/`from_yaml` populate it from path.parent().
- `extract()` branches: if rule.ast.is_some() → ast::ast_match, else parse_data + walk::walk.
- `extensions()` expanded to cover JS/TS/Rust/Python/Go/Kotlin/Bash families.
- Extractor dispatch in crates/index changed from first-match to flat_map over all matching extractors (run-all).

### Production rules added (sprefa-rules.json)
- `cargo-package-name`: extracts `package_name` from Cargo.toml [package].name
- `package-json-name`: extracts `package_name` from package.json name field
- `js-env-var-refs`: select_ast rule -- `process.env.$NAME` pattern in JS/TS, emits `env_var_ref`
- Link rule `dep_to_package`: dep_name → package_name by norm_eq
- Link rule `env_var_binding`: env_var_ref → env_var_name by norm_eq

## Key Insights
- sccache can't cache proc-macros (linker involvement). Handles lib crates well.
- ast-grep's `DeserializeEnv` is re-exported at crate root (`use ast_grep_config::DeserializeEnv`), not from the private `rule` submodule.
- Monomorphization happens at call site, not in the lib crate -- tokio's own artifact IS shareable across projects if feature flags match.
- `cargo clean` is almost never needed. Only for: stale proc-macro fingerprints, linker symbol conflicts after major dep updates, toolchain channel switches.
