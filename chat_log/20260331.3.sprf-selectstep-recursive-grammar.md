# Session: sprf-selectstep-recursive-grammar

## Goal
Make SelectStep recursive so .sprf language lowers directly into existing Rule types, then finalize .sprf grammar design.

## Current State

- SelectStep::Array { item: Vec<SelectStep> } added

- All 216 tests pass, walk.rs handles cross-product for Object entries and per-element Array iteration
- .sprf grammar finalized: function notation tag(body) with paren counting
- PLAN.md updated with final grammar, lowering table, edge cases
- No code written for the parser crate yet

## Problem/Context
Previous plan had a separate Pattern type and _3_match.rs tree matcher -- a parallel path alongside walk.rs. User correctly identified: "if the rust cannot express sprf, then we need to go to drawing board, bc one is the other." Made SelectStep recursive instead, eliminating the parallel path.

## Solution/Approach
1. Made Object variant recursive: entries: Vec<ObjectEntry> with key matching + value sub-chains
2. Added Array variant for element iteration with ancestor carry-forward
3. walk.rs cross-products Object entry results (conjunctive), concats Array element results
4. Grammar uses CSS function notation: json({...}), ast(...), ast[lang](...), re(...), repo(...), branch(...), fs(...)
5. Bare globs are positional context: 3 bare = repo > branch > fs. Otherwise use tags.
6. ** inside json() destructuring = recursive descent (lowers to SelectStep::Any)

## Key Insights
- SelectStep is the universal IR. No escape hatches, no Option<Pattern> fields on Rule.
- Object cross-product: each entry produces capture sets, cross-product gives ancestor carry-forward
- Array: per-element recursion, captures flow down from surrounding context
- json() destructuring keys support: exact, glob, $CAPTURE, $_ wildcard, re:regex, ** (recursive descent)
- Paren-counted delimiters solve the >> escaping problem from previous <<>> design
- Content matchers always tagged (json/ast/re). No inference from body content.
- Bare context globs must be all 3 (repo > branch > fs) or use explicit tags
