# Session: sprefa-js-extractor-e2e

## Goal
Build out sprefa's extraction pipeline: rule engine -> RuleExtractor -> scan pipeline -> JS/TS extractor via oxc. Get end-to-end working: init -> add -> scan -> query.

## Current State
- e2e pipeline working: init, add, scan, query all functional
- RuleExtractor built: compiles RuleSet JSON to CompiledRule, implements Extractor, handles json/yaml/toml via serde, node_path threaded through for anti-unification
- Scan pipeline complete: normalize.rs, files.rs (git ls-files + walkdir + mmap), scanner.rs (rayon parallel parse + batch SQLite tx)
- JsExtractor built: oxc 0.122, extracts ImportPath + ExportName refs with byte spans, wired into CLI alongside RuleExtractor
- Anti-unification spec saved to docs/anti-unification-rule-inference.md
- oxc JS analysis spec saved to docs/oxc-js-analysis.md
- All tests passing (42 rules tests + 7 js tests)

## Key Decisions
- node_path TEXT added to refs table (threaded from MatchResult.path, joined with "/") for anti-unification
- mmap for file reads (memmap2) -- Mmap derefs to &[u8], zero-copy into Extractor::extract
- Scanner takes Vec<Box<dyn Extractor>>, CLI creates RuleExtractor + JsExtractor and passes both
- oxc 0.122: req.span includes quotes -- strip +1/-1 to get bare specifier span
- requested_modules covers all specifiers including re-exports -- no need for indirect_export_entries separately
- RuleExtractor skips rules with no `select` (ast-only rules) until ast engine exists
