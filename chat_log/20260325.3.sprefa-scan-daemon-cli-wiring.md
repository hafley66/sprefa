## Goal
Complete the sprefa scan pipeline: alias extraction, import target resolution, HTTP daemon scan/query endpoints, and CLI daemon delegation.

## Current State
All planned work for this session is complete and committed (ece7b65). The build is clean.

## Problem/Context
The prior session had split crates/scan into index/cache/scan. This session implemented the remaining pipeline features on top of that foundation.

## Solution/Approach
- Two-ref alias design: ImportAlias/ExportLocalBinding linked via parent_key_string_id (no new table)
- Blocking scan queue: tokio::sync::Mutex held for scan duration, 300s timeout returns 503
- CLI daemon delegation: if [daemon].url set and --once not passed, scan/query delegate via reqwest
- git2 tree walk replaces git ls-files subprocess
- 3-tier import resolver in cache/resolve.rs (relative + extension probe, bare specifier via repo_packages, scoped)

## Key Insights
- parent_key_string_id reused for alias links: ImportAlias.parent_key -> ImportName string_id, ExportLocalBinding.parent_key -> ExportName string_id
- Default import proxy span: entry.statement_span.start used as span for "default" string (no physical span in source)
- oxc Argument::StringLiteral is a direct variant (not wrapped), due to inherit_variants! macro
- oxc ModuleRecord.requested_modules does NOT include require() calls -- must AST walk for CommonJS
- Soft scan queue: tokio::sync::Mutex (not semaphore) naturally queues concurrent POST /scan requests
- build_scanner() returns anyhow::Result; cmd_serve calls .ok().map(Arc::new) so daemon starts even without rules file (scan disabled)
